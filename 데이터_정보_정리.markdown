nins(일사량, 목표값)

일반적 가설(모델에 넣기 전 기대)
	•	구름량(cloud_*)와 ceiling → 가장 강한 음의 기여.
	•	uv_idx, vis → 강한 양의 기여.
	•	rain/snow/precip → 음(일시적 거의 0으로 떨어짐).
	•	temp/pressure/humidity → 주로 기상상태(맑음/흐림)의 간접 지표로 사용.
	•	wind 계열 → 보조 지표(전선·폭풍 탐지).

실무 팁 — 전처리·특성공학(우선순위)
	1.	시간 파생변수: hour, sin(hour*2π/24), cos(...), day_of_year, 계절(월) — 태양 궤도·일주기 캡처.
	2.	태양 기하학 파생: 위도·경도 + 시간 → 태양 고도(solar elevation), 일조시간, 태양 방위(azimuth). (매우 중요)
	3.	라그·롤링: nins는 순간값이라 lag(5min/15min/1h)·rolling mean(10~60분) 유용.
	4.	결측 처리: 시간 보간(선형/시간 기반) + 계절·시간대별 보간 권장.
	5.	스케일링/로그: 구름/precip 등은 클리핑·로그 스케일 고려.
	6.	상호작용: cloud * solar_elevation, uv_idx * cloud 등 — 구름이 많을 때 태양각 효과 감소.
	7.	다중공선성 관리: rel_hum/humidity/dew_point, temp계열, wind계열 등은 PCA나 변수 선택.
	8.	범주형 변환: 풍향은 sin/cos 변환 혹은 8방위 범주로 변환.

EDA·중요 분석 방법 (권장)
	•	시간별(시간대·요일·월) nins 평균 플롯 → 일주기/계절성 확인.
	•	cloud, uv_idx, vis 와의 산점도 + 저해상도(heatmap) — 비선형성 확인.
	•	상관행렬(Pearson/Spearman) + 상관계수 히트맵(단, 비선형 관계엔 Spearman 권장).
	•	조건부 분포: nins를 cloud 구간별(예: 00.2, 0.20.6, >0.6)로 나누어 박스플롯.
	•	SHAP/Permutation importance로 모델 기반 중요도 확인.
	•	분해(Decomposition): nins = clear-sky 모델(기하학) * cloud_attenuation + noise 관점으로 접근.

주의사항 / 함정
	•	외부데이터 사용 불가 규정(대회)에 유의 — 하지만 태양각(시간+좌표 계산)은 원천 데이터에서 계산 가능한 파생변수로 허용됩니다(외부 관측 데이터가 아님).
	•	시계열 정렬·리샘플링: train의 nins는 5분 단위인데 입력 변수는 1시간 단위일 수 있음 → 동기화(upsample/downsample) 전략 필요(예: 1h -> 5min 선형/전파 보간 또는 5min nins 집계).
	•	타깃 누수 주의: 발전량(energy) 등은 테스트에 없으므로 train에서 사용 시 재현성/누수 체크 필요.
