# Content Pipeline — 브리핑 작성과 게시 준비

## 1. 목표

이 문서의 목적은 Discord에 게시할 concept/trend 브리핑을 안정적으로 준비하는 기준을 정리하는 것이다.

현재 저장소는 브리핑 발행만 담당하므로, 콘텐츠 파이프라인도 브리핑 본문과 출처 품질에 집중한다.

## 2. 결과물

이 파이프라인의 출력은 아래 두 가지다.

1. 공개용 브리핑
2. 구조화된 출처 목록

## 3. 콘텐츠 종류

### 3.1 Concept Brief
- 예: Perceptron, Attention, Fine-tuning
- 사람이 작성한 markdown 원본 기준

### 3.2 Trend Brief
- 예: 최근 멀티모달 모델 동향
- OpenAlex와 OpenAI를 사용하는 자동 생성 경로
- 반드시 지난 7일 논문 안에서만 고르고, 그 안에서 연구 참고 가치가 높은 논문을 우선 선택

## 4. 작성 단계

```text
1. 주제 선정
2. 출처 수집
3. 브리핑 초안 작성 또는 생성
4. 필수 필드 점검
5. 출처 정규화
6. 게시 전 검토
7. Discord 게시
```

## 5. 필수 데이터

- `briefing_key` 또는 추적 가능한 식별자
- `track`
- `title`
- `one_line`
- `what_happened`
- `why_it_matters`
- `discussion_prompt`
- `sources[]`

## 6. 출처 규칙

- 최소 1개 이상의 출처를 둔다.
- `title`, `url`은 구조화된 필드로 보관한다.
- 가능하면 `published_at`, `source_type`, `canonical_url`도 함께 둔다.
- trend 브리핑은 출처 없는 게시를 허용하지 않는다.

## 7. 품질 규칙

### 브리핑 품질
- 초보자도 읽을 수 있어야 한다.
- 왜 중요한가가 빠지면 안 된다.
- 한 줄 요약과 본문이 서로 모순되지 않아야 한다.

### 출처 품질
- 링크가 실제 문서로 이어져야 한다.
- 제목과 링크가 브리핑 주제를 제대로 뒷받침해야 한다.
- 중복 URL은 정규화 기준으로 관리한다.

### Trend 선정 품질
- 후보는 반드시 지난 7일 논문으로 제한한다.
- 너무 최신이라는 이유만으로 선택하지 않는다.
- 후속 연구에 도움이 되는 방법론, 실험, 재현 단서가 있는 논문을 우선한다.
- 구현 신호와 공개 접근성은 가산점으로만 사용한다.

## 8. 게시 전 체크리스트

- [ ] 제목과 한 줄 요약이 있다
- [ ] 핵심 설명과 중요성 설명이 있다
- [ ] 토론용 질문이 있다
- [ ] 출처가 1개 이상 있다
- [ ] 링크가 실제로 열리는지 확인했다
- [ ] Discord 길이 제한에 맞는지 확인했다
