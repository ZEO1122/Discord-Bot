# Discord Bot Spec — 명령어와 상호작용 설계

## 1. 봇의 역할

이 봇은 단순 공지봇이 아니라 **학습 경험을 운영하는 Discord 앱**이다.

핵심 역할:
- 브리핑 게시
- 퀴즈 제출 UI 제공
- 개인 피드백
- 사용자별 기록 조회
- 운영진 게시/관리

## 2. 권장 채널 구조

```text
#daily-briefs      오늘의 브리핑 게시
#quiz-arena        퀴즈/점수 관련 공개 알림(선택)
#paper-discussion  토론 스레드 또는 심화 토론
#bot-commands      slash command 안내/테스트
#admin-content     운영진 검수 전용
```

## 3. 주요 명령어

### 일반 사용자 명령

#### `/brief today [track]`
- 오늘의 브리핑 조회
- track 예시: `dl-basics`, `nlp`, `cv`

#### `/brief recent [track]`
- 최근 게시물 목록 조회

#### `/quiz latest`
- 가장 최근 열린 퀴즈 진입

#### `/quiz solve [quiz_id]`
- 특정 퀴즈 직접 풀이

#### `/stats me`
- 내 총 응답 수 / 정답 수 / streak / 최근 활동

#### `/leaderboard [period] [track]`
- 기간별 순위 보기

#### `/review wrong`
- 오답노트 보기

### 운영진 명령

#### `/admin publish [content_id]`
- 승인된 초안 게시

#### `/admin reroll [track]`
- 같은 분야에서 다른 초안 재생성

#### `/admin draft list`
- 대기 중 초안 목록 보기

#### `/admin close-quiz [quiz_id]`
- 퀴즈 마감

#### `/admin reveal-answer [quiz_id]`
- 해설 공개

## 4. 브리핑 메시지 구조

브리핑 메시지는 임베드 1개를 기본으로 한다.

### 필드
- 제목
- 한 줄 요약
- 핵심 설명
- 왜 중요한가
- 용어 3개
- 토론 질문
- 출처

### 버튼
- `퀴즈 풀기`
- `힌트 보기`
- `출처 다시 보기`
- `토론 스레드 열기` (선택)

## 5. 퀴즈 UI 원칙

### 객관식
- 버튼 또는 select 메뉴
- 정답 판정이 빠르고 안정적
- 랭킹 반영에 유리

### 단답형
- modal 입력창 사용
- 규칙 기반 정규화 후 판정
- 불확실하면 부분 점수 또는 재검토

### 서술형
- 초반에는 점수화하지 않거나 참여 점수만 반영
- 운영진 검토 또는 LLM 보조 채점은 2차 단계

## 6. 피드백 UX

### 공개 채널
공개 채널에는 아래 수준만 노출한다.
- 총 응답자 수
- 정답률
- 마감 여부
- 해설 공개 여부

### 개인 피드백
개인에게는 아래 정보를 제공한다.
- 정답/오답
- 제출한 답
- 정답 또는 허용 답
- 해설
- 다시 풀기 가능 여부
- 현재 내 누적 통계

## 7. 퀴즈 상태

권장 상태값:
- `draft`
- `scheduled`
- `open`
- `closed`
- `revealed`
- `archived`

## 8. 점수 규칙 예시

### MVP 점수
- 첫 시도 정답: 10점
- 재시도 정답: 4점
- 오답 제출: 1점 참여점수
- 토론 참여: 1~2점(선택)

### 순위 지표
- 총점
- 첫 시도 정답 수
- 연속 참여 일수
- 분야별 점수

## 9. 권한 설계

### 일반 사용자
- 브리핑 조회
- 퀴즈 제출
- 내 기록 조회
- 리더보드 조회

### 운영진
- 게시 승인
- 퀴즈 마감
- 답 공개
- 재게시
- 로그 조회

## 10. 예외 상황

### 중복 제출
- 기본 정책을 명확히 정한다.
- 예: 1차 MVP는 재시도 허용, 첫 시도 정답률은 별도 집계

### 마감 후 제출
- 점수 없음 또는 비공식 제출 처리

### 게시물 삭제/수정
- `publish_logs` 와 `discord_message_id` 기준으로 재동기화 필요

## 11. Discord 복잡도 줄이는 방법

- 공개 메시지는 최대한 단순하게 유지
- 개인 결과는 개인에게만
- 통계는 slash command에서 조회
- 관리 기능은 admin 역할에만 노출

## 12. MVP 체크리스트

- [ ] 브리핑 임베드 게시
- [ ] 퀴즈 버튼 동작
- [ ] 객관식 제출 및 저장
- [ ] `/stats me`
- [ ] `/leaderboard`
- [ ] `/admin publish`
- [ ] 마감/정답 공개
